## 2. パターン別分析

### 2-1. ヒーロー（トップビジュアル）セクション

#### ぶいすぽっ！
- **大型カルーセル方式**。7人分のメンバービジュアルが自動回転。
- 背景に**ドット柄テクスチャ**を敷き、立ち絵が映える奥行きを演出。
- タグライン「GAMING GIRLS ARE COOL&CUTE」を大きなテキストで前景に置く。
- 各ビジュアルは 1920×1500px 相当の大版。
- キャラが背景ごと「1キャラ = 1枚のビジュアル」として完結している。

#### ゲームサイト共通
- メインキービジュアルが**フルブリード**（画面端まで塗りつぶす）。
- ページロード時に**カーテンアニメーション**（ページが開くように展開）を使うサイト多数。
- ヒーロー部でのテキストは最小化し、ビジュアルに主役を渡す。

#### 重要なパターン：「前景立ち絵 + 背景バストアップ」
- 「メメントモリ」でよく見られる手法。
- 手前に**透過PNG立ち絵**を配置し、背後にキャラのバストアップ/アート画像をぼかして敷く。
- 2枚の画像がキャラクターの**存在感と内面性**を両立させる。

### 2-2. キャラクター一覧ページ

#### ウマ娘（グリッド + ソート型）
- **均一サイズのグリッドカード**を並べるシンプルな一覧。
- 各カードに：キャラクター画像・名前（日英）・声優（CV）・「View more」リンク。
- **ソートバー**付き（デフォルト / 名前順 / 身長順）。
- 検索バーも実装 → 名前で絞り込み可能。
- カード背景は控えめで、**キャラクター画像が主役**。
- ページ下部に「pagetop」ボタン。

#### プリコネR（グループ分け + 縦スクロール型）
- **ギルド（チーム/組織）別**にセクションを分けてキャラを配置。
- アンカーリンクナビで各グループへジャンプ可能。
- 縦スクロールで全員を一覧できる設計。
- キャラ間に**均等な間隔**と統一感。

#### にじさんじ / ホロライブ（世代・ユニット分け型）
- 「期・ユニット」などのグループ分けを最上位構造に据える。
- 各メンバーカードは小〜中サイズで、SNSリンク付き。
- ホバーで名前や簡単なプロフが浮き上がる効果が一般的。

#### Paradox Live（チーム別 + ロゴ重視型）
- チームごとに**ロゴ画像 + ビジュアル**をセットで表示。
- チームのビジュアルアイデンティティを前面に出す。
- キャラ個人よりも**グループとしての世界観**を優先した構成。

### 2-3. 個別キャラクター詳細ページ

#### 基本構成（多くのサイトで共通）
1. **フルブリードのキャラビジュアルヒーロー**（立ち絵 + キャラカラー背景）
2. **プロフィール情報**（名前、年齢、声優、一言など）
3. **ストーリー・紹介文**
4. **スキル・強み一覧**

#### 付加コンテンツ（プラスα系サイト）
- **相関図**：他キャラとの関係を図示
- **ショートストーリー**：テキストで掘り下げ
- **インタビューボイス**：キャラのセリフをボイス付きで
- **所持品・設定画**：キャラを深掘りするビジュアル

#### ギミック系（インタラクティブ型）
- 「Other side」ボタンでモノクロ/カラーを切り替え → 表と裏の顔を演出
- スクロールに連動して立ち絵がパーツごとに登場するパラレックス効果
- キャラ固有の「秘密」情報をユーザーアクション後に解放

### 2-5. タイポグラフィ傾向

- **世界観に合ったフォント選定**がキャラサイトの最重要要素のひとつ。
  - ウマ娘：「わんぱくルイカ08」（元気でチャーミングなゴシック体）
  - ダーク系作品：セリフ体 or 細身の欧文フォント
- 日本語と英語のバランスをとる（両言語表記のサイトも多い）。
- タイトルロゴは**「作品の顔」**として独自フォント/カスタムロゴを使用。
- 本文は読みやすさ優先の標準フォント（ヒラギノ角ゴ、Noto Sans等）。

---

### 2-6. カラー・ダーク/ライトテーマ傾向

- **ダーク系が主流**（黒・ディープネイビー・チャコール背景）。
  - キャラクターのカラフルな立ち絵が引き立つ。
  - シリアス・ドラマティックなトーンに合う。
- **キャラ固有のアクセントカラー**を各ページ/カードに適用するのがスタンダード。
  - CSS カスタムプロパティ（`--accent`）での動的色変更が有効。
- ホワイト背景を使う場合は、キャラカラーをカード背景に使い差別化。
- グラデーションを活用（背景色 → 透明、キャラカラー同士のミックスなど）。

---

### 2-7. モバイル対応

- **縦スクロール型の一覧**が主流（横スクロールはカルーセルのみに限定）。
- スマホでは 1〜2カラムのグリッド。
- キャラカードを**縦長レイアウト**（立ち絵上部 + 情報下部）にすることでモバイルでも見やすい。
- ヒーローセクションはキャラが切れないよう `object-position` で中心部を維持。
- ナビゲーションはハンバーガーメニューが標準。

## 3. まとめ：キャラクタードリブンなサイトの共通要素

キャラが「主役」として機能しているサイトに共通するデザイン上の判断：

1. **画像が大きい** — カードのうち画像が占める面積が 60〜80%
2. **テキストを削ぎ落とす** — 一覧では名前・キャッチフレーズ・1〜2行の情報だけ
3. **色がキャラを語る** — 背景・ボーダー・グロー効果でキャラカラーを全面活用
4. **ホバーが「見たい」を作る** — 一覧では画像拡大 + 名前浮き出しで詳細への誘導
5. **詳細ページで世界観を深める** — 1キャラ = 1ページとして「読める」構成
6. **動きが存在感を与える** — ロード演出・スクロール連動・Live2D等で「生きている」感
