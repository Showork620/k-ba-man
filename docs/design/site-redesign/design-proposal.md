## 方針：「円卓の劇場」としてのサイト

現状の課題は「データポータル」に見えてしまうことだ。  
目指すのは**「競馬予想の円卓ドラマが繰り広げられる劇場の入口」**。

10人の予想屋は、それぞれの流派と人生を背負って円卓に座っている。  
サイトに訪れた人が「この人はどんな人なんだろう」と思い、  
レース結果が出たときに「龍之介当てた！！」と興奮できる——  
そういう感情的な関わりをデザインで作る。

---

## ビジュアルトーン

### テーマ：「深夜の円卓、かすかな光」

- **ベース**: ディープチャコール（`#0e0e12`〜`#1a1a22`）のダーク背景
- **テクスチャ**: 微細なノイズグレインまたは布目テクスチャ。安っぽくならない程度に薄く。
- **アクセント**: 各キャラ固有のカラーをグロー/ボーダー/グラデで活用
- **ゴールド**: 競馬・格式感のアクセントとして `#c8a96e` を見出しやラインに使う
- **テキスト**: オフホワイト（`#e8e4dc`）。純白より落ち着きがある

### フォント方針

| 用途 | 推奨 | 理由 |
|------|------|------|
| 見出し（日本語） | 源ノ明朝 / Noto Serif JP Bold | 格調・ドラマ感 |
| 見出し（英語・数字） | Playfair Display / Cormorant Garamond | セリフ体でクラシカル感。Cormorant はより細身・格式感強め |
| ナビゲーション | Playfair Display（大文字、letter-spacing: 0.2em） | `CHARACTERS` `RANKING` を格調ある印象に |
| 本文 | Noto Sans JP | 可読性優先 |
| キャッチフレーズ・強調 | 既存フォントのウェイト変化で対応 | まず既存で試す |

> Playfair Display と源ノ明朝の組み合わせは「競馬の格式 × ドラマのリッチさ」を体現する。  
> Cormorant Garamond は Playfair よりさらに細身で「馬券の印刷物」的な格式感。大見出し専用に限定すること（本文への使用は可読性が下がる）。

## ページ別 UI 設計案

---

### トップページ (Overview)

#### 現状の課題
- ヒーロー左半分がテキストブロック → キャラが「補足要素」に見える
- キャラグリッドが小さすぎる
- 集合ビジュアルは右側にあるが、スクロールすると消える

#### 改善案：フルブリードキャラヒーロー + スライダー

```
[ヒーロー — フルスクリーン]
  背景: ダーク + キャラカラーの薄いグラデーション
  中央: lineup-real-preview.png を大きく（または各キャラの立ち絵を横並び）
  前景オーバーレイ（下部から）:
    ブランドロゴ + キャッチコピー
    「10人を見る」「最新成績を見る」CTA

[スクロール後: サマリーストリップ（現状維持）]
  成績数値タイルが4枚

[キャラクター予告セクション]
  横スクロールカルーセル OR 大きいグリッド
  カードは縦長、立ち絵が画面の高さの 50〜60% を占める

[最新レースセクション（小さく）]
```

**ヒーローの2案：**

**A案（静的フルブリード）**:  
集合ビジュアル `lineup-real-preview.png` をフルスクリーンに広げ、  
下部から暗いグラデーションで文字エリアを作る。  
シンプルで実装コスト低。

**B案（キャラクタースライダー）**:  
10人を1人ずつフルスクリーンで表示。  
自動再生または手動でスライドを切り替え。  
各スライドにキャラカラーの背景、セリフ、名前を表示。  
没入感が高い。ぶいすぽっ！型。

### キャラクター一覧ページ (Characters)

#### 現状の課題
- カードが小さく、数値情報が先に目に入る
- キャラクター性よりも「成績表」に見える

#### 改善案：縦長カード + ホバー展開

```
[キャラクター一覧]
  レイアウト: 4〜5 カラムグリッド（モバイル: 2カラム）
  
  [各カード — 縦長]
    上 60%: キャラクター立ち絵（object-fit: cover、顔が見えるよう top 寄り）
    中段: シートナンバー + 名前（大きめ）
    下段: キャッチフレーズ（alias）
    
    ホバー時:
      - カードが少し浮く（translateY -4px + shadow 強化）
      - キャラカラーのグローが縁に出る
      - 流派テキストがフェードイン
      - カード下部に「詳細を読む →」が現れる
    
    カード背景:
      通常: ダーク（`#16161e`）
      ホバー: キャラカラーを 10〜15% 透明度で背景に敷く
```

**カードのサイズ感（目安）:**
- PC: 幅 200px、高さ 320px（縦長）
- 立ち絵エリア: 高さの 65%
- 情報エリア: 高さの 35%

**フィルタリング（任意）:**
- 「全員」「本命派」「穴党」などのタグで絞り込み（実装は後回しでも可）

### 個別キャラクター詳細ページ (Character Detail)

#### 現状の課題
- プロフィールヒーローでキャラ画像が右に小さく配置されている
- 背景が白っぽく、「データページ」の雰囲気が続く
- セリフ（quote）が blockquote として埋もれている

#### 改善案：フルブリードキャラヒーロー + レイヤード演出

```
[キャラヒーロー — フルブリード]
  背景: キャラカラーを使った縦グラデーション（上: キャラカラー 30% → 下: ダーク）
  
  [レイヤー構成]
    最奥: キャラカラーグラデーション背景
    中層: キャラのバストアップ画像（*-real.png）をぼかして大きく配置
    前景: 透過PNG立ち絵を中央〜右に配置（高さ 80vh 相当）
    テキストレイヤー（左寄り）:
      - シートナンバー（小さく、ゴールドで）
      - 名前（大きく、明朝体）
      - alias（英語表記のように）
      - セリフ（blockquote — キャラカラーの縦線付き、斜体）

[スクロール後: プロフィールセクション]
  背景: ダーク（`#0e0e12`）
  2カラム（PC）:
    左: intro + drama テキスト
    右: strengths タグ + weakness

[成績セクション]
  背景: 少し明るいダーク（`#161620`）
  スタット4タイルは現状のデザインを踏襲しつつ、キャラカラーでラインを引く

[成績履歴テーブル]
  現状維持でOK。ただし行ホバーでキャラカラーハイライト。
```

**セリフ（quote）の見せ方：**  
現状は `<blockquote>` に埋もれているが、**キャラヒーローエリアで最も大きく見せる要素**にする。  
フォントサイズ 1.5〜2rem、キャラカラーの縦線＋斜体で「その人の声」として際立てる。

**切り替えギミック（任意 / 後フェーズ）:**  
通常絵（`*.png`）↔ リアル絵（`*-real.png`）の切り替えボタン。  
「素顔を見る」的なコピーで、キャラへの興味を深掘りするギミックにできる。

## インタラクション優先順位

実装コスト × 効果で優先度を整理する。

| 優先度 | インタラクション | 効果 |
|--------|-----------------|------|
| 高 | カードホバーでグロー + 浮き上がり | キャラを「触れたい」に変える |
| 高 | キャラヒーローのフルブリード化 | 一番インパクトが大きい |
| 高 | セリフを大きく前面に出す | キャラの個性が伝わる |
| 高 | スクロール連動入場アニメーション（CSS native） | 画面に入った瞬間に要素が現れる没入感 |
| 中 | ヒーロースライダー（B案） | 10人を順に見せる没入感 |
| 中 | ページロードのカーテン演出 | ゲームサイト的ファーストインプレッション |
| 中 | 成績数値の滑らかカウントアップ | データが「生きている」感を与える |
| 中 | ページロードのフェードイン | プロフェッショナル感 |
| 低 | 通常絵↔リアル絵切り替えギミック | 深掘り層向け |
| 低 | キャラアイコンクリックで音声再生 | 声優ボイスがあれば強烈な個性付け（要コンテンツ） |
| 低 | Live2Dアニメーション | コスト大、後フェーズ |

## CSS 設計の変更点（実装時のメモ）

フォーク元のコードベース（バニラ JS SPA）を前提とした実装ヒント。実装時に参照すること。

- **カスタムプロパティの活用**: 現状 `--accent` が既に各キャラに定義されている。これをグロー・背景・ボーダーに広く使う。
- **グラデーション背景**: `linear-gradient(to bottom, color-mix(in srgb, var(--accent) 30%, transparent), #0e0e12)` のパターンを詳細ページヒーローに使う。
- **ガラスモーフィズム（ヘッダー）**: `backdrop-filter: blur(12px)` + 半透明背景。
- **カード縦長化**: `.character-card` の `aspect-ratio` を `3/4` または `2/3` に変更。
- **スクロール連動アニメーション（CSS native）**: `animation-timeline: scroll()` または Intersection Observer + `@keyframes` で JS ライブラリ不要のスクロール演出。カードや見出しをスクロール入場時に `opacity: 0 → 1` + `translateY(20px → 0)` でフェードイン。
- **数値カウントアップ**: 成績スタット数値（回収率・的中数等）はページロード時に 0 からカウントアップ。`requestAnimationFrame` で 800〜1000ms かけて実装し「データが動いている」感を演出。
- **セクション固有背景**: 提案〜成績など各セクションに `background-image` を個別設定し、スクロール時の雰囲気変化を作る（メメントモリ参考）。

## モバイル対応方針

- **ヒーローセクション**: キャラ顔が見えるよう `background-position: center top` を維持
- **キャラカード**: 2カラムグリッド（1カラムは圧迫感が出る）
- **詳細ページ**: PC の左テキスト + 右画像 → スマホでは画像先行 + テキスト後
- **ランキング行**: モバイルでは成績数値を折りたたみ、名前+カラーボーダーだけ見せる

## 追加調査知見（2026-06-17 第2次調査）

初回提案後、以下のカテゴリで追加調査を行い、補足知見を統合した。

### A. メメントモリ公式サイトから学ぶ

メメントモリ（`mememori-game.com`）は現行提案で「前景/背景レイヤー切り替え」参考として言及されていたが、実際のサイトから以下の追加パターンを確認した。

**1. 「Now Loading...」ローディング画面演出**  
ページ遷移に必ず挟まるロード演出が、VTuber/ゲームサイトの「劇場の幕が上がる」感を作っている。  
k-ba-man への転用: 初回アクセス時のみ、`0.8〜1.5秒` のロード演出（ロゴ + スピナー）を設ける。  
実装コストは低い（CSS animation のみ）が、ファーストインプレッションに与える効果は高い。

**2. キャラアイコン × 音声インタラクション**  
各キャラアイコンをクリックするとキャラ固有のラメント（楽曲）が流れる仕組み。  
k-ba-man 用の転用案：キャラクター一覧カードのホバー/クリック時にキャラの「口癖セリフ」を表示する演出はすでに提案済みだが、**ボイスデータ（CV）が用意できれば** 音声再生まで発展させる余地がある。現時点ではテキストセリフ演出を優先し、音声は後フェーズ扱いが妥当。

**3. セクション固有の背景画像**  
「GALLERY」「STORY」「NEWS」など各セクションで背景ビジュアルを切り替え、長いページでも飽きさせない工夫。  
k-ba-man 用：キャラ一覧・成績・レースの各セクションで、微妙に異なるテクスチャ/グラデーションを使う。CSS background が異なるだけで、セクション間の「変化感」が生まれる。

**4. 大文字間隔ナビゲーション**  
`_N__E__W__S_` のように文字間隔を広げた大文字表記。格式感・アート感が増す。  
k-ba-man では `CHARACTERS` `RANKING` `RACES` などのナビラベルに `letter-spacing: 0.2em; text-transform: uppercase;` を適用すると競馬・格式感が強まる。

### B. 競馬・スポーツ予想UIからの知見

「Midnight Shadow — Horse Profile Web UI Design」（Figma コミュニティ）など、競馬特化のダークUIコンセプトに共通する要素：

**ミニマル×エレガント のバランス**  
VTuber系サイトが「賑やかな印象」なのに対し、競馬・エクエストリアン系のプレミアムUIは  
「余白を多く取る」「線を細く保つ」「情報を絞る」傾向にある。  
k-ba-man の差別化ポイント：**10人のキャラを賑やかに見せながらも、競馬らしい格式を背景に宿らせる**ことで、VTuberサイトとの差別化ができる。

**具体的な格式演出レシピ：**
- 細い水平ルール（`border: 1px solid rgba(200,169,110,0.3)`）で情報を区切る
- セクション見出しに細い上下ラインを添える（テーブルのヘッダー風）
- 数値（回収率・的中率）は大きく表示するが、周囲に余白を多めに取る
- 馬券の「票」「倍率」をイメージした数値スタイリング

### C. 予想市場UXからの知見（信頼感設計）

予想・ベッティング系UIの研究（avark.agency）から、k-ba-man のデータ表示に転用できるパターン：

**1. プログレッシブディスクロージャー（段階的情報開示）**  
- トップページ → キャラ成績の「サマリー」だけ（的中率・回収率の数字2〜3個）
- キャラ詳細ページ → スタット4タイル + 成績履歴テーブル  
- （将来）専門ページ → 流派ごとの分析データ、相関マトリクス等  
「見せすぎない」ことで、キャラクター性を前面に保ちながら、深掘りしたい人には道を作る。

**2. 数値の滑らかアニメーション**  
予想市場では確率が急変するとユーザーが不安になるため、200〜300ms かけてなめらかに変化させる。  
k-ba-man では成績スタットのカウントアップに同じ原則を適用する（→ CSS設計セクション参照）。

**3. 根拠の可視化（信頼シグナル）**  
予想屋キャラの「流派タグ」（血統派・展開読み・穴党）を一覧で小さく表示することは、  
「この人はどういう根拠で予想している人か」を視覚化する信頼シグナルになる。  
キャラカードのホバー時に流派をフェードインする案（既存提案）はこの観点でも有効。

### D. CSS スクロール駆動アニメーション（技術補足）

2024〜2025年に Chrome / Safari でネイティブ実装が進んだ **CSS Scroll-Driven Animations** は、  
k-ba-man のバニラ JS SPA で追加ライブラリなしに使える。

```css
/* キャラカードがスクロールで入場する例 */
@keyframes card-enter {
  from { opacity: 0; transform: translateY(20px); }
  to   { opacity: 1; transform: translateY(0); }
}

.character-card {
  animation: card-enter linear both;
  animation-timeline: view();          /* スクロール位置に連動 */
  animation-range: entry 0% entry 40%; /* 画面に入り始めてから 40% 進むまで */
}
```

ただし Safari の対応状況（2025年時点で部分対応）を考慮し、  
`@supports (animation-timeline: scroll())` でフォールバックを書くか、  
AOS.js（~2KB）を使う二択になる。AOS.js の方が現時点では安定。

### E. フォント方針の補足

既存提案（Playfair Display + 源ノ明朝）に加え、**暗背景向けのカラー値**を記録する。

| 組み合わせ | テキスト色 | 背景色 | 特徴 |
|-----------|-----------|--------|------|
| Playfair Display + Source Sans Pro | `rgb(240, 214, 187)` | `#353535` | ウォームな暗背景で実績あり |
| Playfair Display + Noto Sans JP | `#e8e4dc`（既存提案） | `#0e0e12` | 本案のベース設定 |
| Cormorant Garamond + Noto Sans JP | `#e0d8cc` | `#12121a` | よりフォーマル・格式感強め |

> Cormorant Garamond は Playfair Display よりさらに細身でクラシカル。  
> ナビやラベル以外の大見出しに使うと「馬券の印刷物」的な格式感が出る。  
> 本文可読性が下がるため、見出し専用に限定すること。
