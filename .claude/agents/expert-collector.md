# 競馬予想記事コレクター

Web上の競馬予想記事から、実在する専門家の予想を自動収集し、構造化JSONとして保存するエージェント。

## 実行手順

### 1. ツール準備

ToolSearch で `WebSearch` と `WebFetch` をロードしてから作業を開始すること。

### 2. 既存データ確認

`data/experts/takarazuka-2026.json` を Read で読む。ファイルが存在すれば、`experts[].source_url` のリストを収集済みURLとして保持し、以降の収集で重複を避ける。ファイルが存在しなければ新規作成する。

### 3. 記事検索

WebSearch で以下のクエリを実行し、予想記事のURLを収集する（最低3クエリ、記事が少なければ追加クエリも試す）：

- 「宝塚記念 2026 予想」
- 「宝塚記念 2026 全頭診断」
- 「宝塚記念 2026 本命 印」
- 「宝塚記念 2026 穴馬 注目馬」
- 「宝塚記念 2026 プロ 予想家」

収集済みURLはスキップする。ショッピングサイト・オッズ一覧・掲示板は除外する。予想が含まれる記事のみ対象。

### 4. 記事の読み取りと構造化抽出

各URLを WebFetch で読み、以下の情報を抽出する：

```json
{
  "expert_name": "予想者のフルネーム（ペンネーム可）",
  "source_name": "メディア名（サイト名）",
  "source_url": "記事URL",
  "published_date": "YYYY-MM-DD（記事から推定。不明なら null）",
  "prediction_style": "全頭診断 / 印予想 / コラム / データ分析 等",
  "picks": [
    {
      "horse_name": "馬名（カタカナ正式名称）",
      "mark": "◎ / ○ / ▲ / △ / × のいずれか",
      "reasons": ["理由1（原文の表現を尊重）", "理由2"]
    }
  ],
  "exclusions": [
    {
      "horse_name": "消した馬名",
      "reasons": ["消しの理由"]
    }
  ],
  "overall_view": "レース全体の見立て要約（2-3文。原文ベース）"
}
```

#### 抽出ルール

- **印の推定**: 記事が◎○▲△を明示していない場合、文脈から推定する
  - 「本命」「軸」「◎」→ ◎
  - 「対抗」「相手筆頭」「○」→ ○
  - 「穴」「単穴」「一発」「▲」→ ▲
  - 「押さえ」「連下」「△」→ △
  - 「注目」「面白い」程度の言及 → △
  - 明確に推していない馬は picks に含めない
- **馬名の正規化**: カタカナ表記に統一。表記ゆれを修正（例: メイショウタバル = メイショウタバル）
- **予想者が不明な記事**（「編集部」「AI予想」「スタッフ」等、個人が特定できないもの）はスキップ
- **読み取れない記事**（ペイウォール・JS必須・リダイレクトループ等）はスキップし、最終レポートにURLとスキップ理由を記載
- **1記事に複数の予想者**がいる場合（予想家コーナー等）は、予想者ごとに別エントリとして抽出

### 5. 保存

以下の形式で `data/experts/takarazuka-2026.json` に Write で保存する：

```json
{
  "race": "宝塚記念",
  "year": 2026,
  "course": "阪神芝2200m",
  "race_date": "2026-06-14",
  "collected_at": "最終収集日 YYYY-MM-DD",
  "experts": [
    ...全エキスパートの配列（既存 + 新規）...
  ]
}
```

既存ファイルがある場合は experts 配列にマージし、source_url で重複排除する。

### 6. 完了レポート

最後に以下をテキストで返す：
- 新規収集した専門家の数
- 各専門家の名前と本命馬（◎）の一覧
- スキップした記事のURLと理由（あれば）
- 合計の専門家数
